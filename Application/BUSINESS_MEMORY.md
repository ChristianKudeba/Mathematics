## What the product is (as of 2026-05-02)

A SaaS platform where mathematicians (and eventually physicists, chemists, anyone whose work admits autonomous AI reasoning) configure **custom AI research agents**. Each agent is built from:
- Tools (e.g., Lean theorem prover, computer algebra, web search, citation lookup)
- Prompts the researcher engineers
- A scope of problems the agent is meant to think about

Agents run autonomously on long horizons. Their outputs feed into a **public, growing "map of mathematics"** — a knowledge graph that is:
- Maintained over time
- Verified by Lean code where possible
- Reviewed by human experts otherwise

There will be **template agents** the platform offers as starting points — e.g., a "theory-builder" agent, a "problem-solver" agent. The user envisions many such templates.

When a researcher's agent makes a **breakthrough**, the researcher is **notified**. Attribution defaults to the researcher (since they engineered the prompt). **Crediting the platform is optional** — the user wants this to make adoption appealing.

The output is **publicly accessible**. The customers paying are the researchers / departments / private labs. The public reads the knowledge graph for free.

## Business model

- Free public read access to the knowledge graph.
- **Paying customers**: individual mathematicians, math departments, universities, private research labs.
- The user explicitly likens this to Mathematica / Maple in business model — i.e., a tool that institutions and serious users pay for.
- Comparable: site licenses, individual subscriptions.

## Go-to-market plan (current)

The first concrete step is **talking to mathematicians**. This serves a double purpose:
1. **Advertise** — get the idea in front of potential users early.
2. **Discover** — figure out what to build based on actual research needs.

The user has noticed a recurring sentiment in the math community: mathematicians are excited about the **potential** of ChatGPT and similar tools, but report that **results are limited and disorganized**. The product is positioned to address this gap — structured, verified, persistent output, organized into a navigable knowledge graph.

## Generalization roadmap

Math first, but the platform is intended to generalize to:
- Physics
- Chemistry
- Any field where autonomous reasoning agents can produce original results

The architecture (custom agents + tools + verifiers + public knowledge graph) is meant to be domain-agnostic.

## Key product differentiators (as articulated so far)

1. **Researcher-owned custom agents** — not a generic chatbot; you build your own thinker.
2. **Verification loop** — Lean for mechanical verification, humans for the rest.
3. **Cumulative public knowledge graph** — output persists, accumulates, and is interlinked over time.
4. **Attribution to the prompter** — researcher gets credit.
5. **Optional platform credit** — flexibility lowers adoption friction.
6. **Breakthrough notifications** — researcher hears about wins from their agent immediately.
7. **Template agents** — theory-builder, problem-solver, etc. as starter kits.
